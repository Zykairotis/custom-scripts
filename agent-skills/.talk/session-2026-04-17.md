## 9. install.sh — Architecture & Logic

### Modes
```bash
bash install.sh              # full interactive install
bash install.sh --check      # audit only, no changes
bash install.sh --env-only   # only prompt for env vars
bash install.sh --tools-only # only install tools
```

### Steps executed
1. **Guard checks** — bash 4+ required, config.yaml must exist
2. **OS detection** — `uname -s` + `/proc/version` WSL check
3. **Package manager detection** — tries candidates in order, first found wins
4. **Copy skills** — copies all folders from repo to `~/.agents/skills/`, skips existing
5. **Tool check loop** — 4-method check per tool, collects missing lists
6. **Install offer** — asks user, installs missing tools, runs `post_install` if defined
7. **Python packages** — checks 4 groups via `python3 -c "import ..."`, offers install
8. **Env var prompts** — skips already-set vars, hidden input for secrets, auto-generates keys
9. **Auth reminders** — prints all manual auth steps
10. **Runtime reminders** — prints external service requirements
11. **sync-skills.sh** — offers to run `~/.agents/sync-skills.sh` to symlink to all harnesses
12. **Shell profile** — offers to add `source ~/.agents/skills.env` to `~/.zshrc` or `~/.bashrc`

### OS → Package Manager mapping
| OS | Candidates (in order) |
|---|---|
| Linux (Arch/WSL) | pacman → apt → brew |
| Linux (Debian/Ubuntu) | apt → brew |
| macOS | brew |
| Windows (Git Bash) | choco → winget → scoop |

### Tool install fallback chain
```
PM-specific command
  → any (universal curl script)
    → npm (if node available)
      → cargo (if cargo available)
        → warn "no install method found"
```

### YAML parsing
No external YAML library used. All parsing done with inline Python3 heredocs using `re` module. Handles:
- `tools.<name>.check`
- `tools.<name>.install.<pm>`
- `tools.<name>.post_install`
- `tools.<name>.required`
- `env_vars.<NAME>.<field>`
- `auth_steps` block
- `runtime_requirements` block

Saved to `~/.agents/skills.env`. Format:

Sourced by adding to `~/.zshrc` or `~/.bashrc`.

## 13. Decisions Made

1. **Flat skill structure** — all 176 skills in one flat `agent-skills/` folder, no subfolders by source. Easier for `sync-skills.sh` to pick up.

2. **Full folder copy** — copy entire skill folder (scripts, templates, configs, references), not just `SKILL.md`. Skills like `backtesting-trading-strategies` have Python scripts, `planning-with-files` has shell scripts and templates.

3. **config.yaml as source of truth** — install.sh reads config.yaml via inline Python3 heredocs. No external YAML library dependency. Means anyone can edit config.yaml to add tools/vars without touching the script.

4. **4-method tool detection** — `command -v` + `which` + package manager query + direct execution. Handles edge cases where a tool is installed but not in PATH, or installed via a different mechanism.

5. **`~/.agents/skills.env` for env vars** — separate from `~/.zshrc` to keep secrets out of the main shell config. Sourced on demand.

6. **sync-skills.sh integration** — rather than duplicating the sync logic, call the existing script. It handles relative symlinks correctly for all 34 harnesses.

7. **Bash 4+ guard** — macOS ships bash 3.2 which lacks `declare -A`. Rather than rewriting to avoid associative arrays, added a clear error message directing macOS users to `brew install bash`.

8. **Auto-generate secrets** — `AGENT_BROWSER_ENCRYPTION_KEY`, `FLAGS_SECRET`, `CRON_SECRET`, `AUTH0_SECRET`, `AUTH_SECRET`, `GITHUB_WEBHOOK_SECRET`, `DRAIN_SECRET` are all auto-generated via `openssl rand` if not already set. User doesn't need to manually create these.

9. **OS-specific tool skipping** — `scrot` (Linux screenshot tool) and `nmap` (not useful on Windows) are skipped when the OS doesn't support them, rather than failing or warning unnecessarily.

10. **Deduplication by folder name** — when collecting skills from multiple sources, if a folder name already exists in `agent-skills/`, skip it. First source wins. Priority: `~/.agents/skills/` > openfang bundled > vercel > google-drive > factory > superpowers.
